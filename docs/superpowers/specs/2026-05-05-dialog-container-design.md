# DialogContainer — shared modal shell

**Date:** 2026-05-05
**Status:** Design approved, implementation pending
**Workstream:** Cleanup PR (CSJTODO line 68 — DialogContainer follow-up)

## Problem

Five ZA modals duplicate the same modal shell pattern (backdrop + panel + header) with inconsistent styling and inconsistent accessibility:

| Modal | Backdrop | Teleport | ARIA roles | ESC key | Close X |
|---|---|---|---|---|---|
| `ZA/Savings/TfsaContributionModal.vue` | `bg-horizon-900/60` | no | no | no | no |
| `ZA/Investment/ZaInvestmentForm.vue` | `bg-horizon-900/60` | no | no | no | no |
| `ZA/Retirement/ZaContributionModal.vue` | `bg-horizon-900 bg-opacity-50` (old syntax) | no | no | no | no |
| `ZA/Retirement/ZaRetirementFundForm.vue` | `bg-horizon-900 bg-opacity-50` (old syntax) | no | no | no | no |
| `ZA/Protection/ZaProtectionPolicyModal.vue` | `bg-horizon-500/40` | yes | yes | yes | yes |

`ZaProtectionPolicyModal.vue` is the accessibility gold standard; the other four lack `<teleport>`, `role="dialog"`, `aria-modal`, ESC handling, and a visible close button. Refactoring onto a single shared shell brings the four laggards up to the gold standard, removes ~20–30 lines of boilerplate per modal, and locks in one canonical backdrop / panel pattern.

## Goals

1. One shared modal shell at `resources/js/components/Shared/DialogContainer.vue` matching the placement of `Shared/ConfirmModal.vue` (already extracted in May session 2).
2. Refactor the 5 ZA modals onto it in **one atomic PR** (per CSJTODO scope).
3. Standardise on the gold-standard a11y behaviour (teleport, ARIA, ESC, close X) for every modal that adopts it.
4. Single backdrop colour: `bg-horizon-900/60`.
5. No regressions in any of the 5 modals — Playwright open / fill / submit / cancel / ESC / backdrop-click verified per modal.

## Non-goals

- Refactoring UK modals onto DialogContainer (out of scope; `Shared/` placement makes future UK adoption possible without committing to it now).
- Absorbing or replacing `Shared/ConfirmModal.vue` — it's a different shape (yes/no confirmation prompt, not a form host) and stays separate.
- Extracting a shared `Tabs.vue` (deferred — only 2 divergent tab-bar patterns exist today; revisit when WS 1.6b adds a third).
- Adding `closeOnBackdrop` / `closeOnEscape` / `#header` / `#footer` slots — none of the 5 modals need them; YAGNI.

## API

### Usage

```vue
<DialogContainer
  v-if="showModal"
  title="Record a TFSA contribution"
  :subtitle="`Tax year ${taxYear || '2026/27'}`"
  size="lg"
  @close="showModal = false"
>
  <form @submit.prevent="handleSubmit">
    <!-- form fields -->
    <!-- inline submit / cancel buttons stay in the form -->
  </form>
</DialogContainer>
```

### Props

| Prop | Type | Default | Notes |
|---|---|---|---|
| `title` | `string` | required | Rendered as `<h2>`, becomes `aria-labelledby` target. |
| `subtitle` | `string \| null` | `null` | Small text under title, optional. |
| `size` | `'sm' \| 'md' \| 'lg' \| 'xl' \| '2xl'` | `'lg'` | Maps to Tailwind `max-w-*` classes. |

Size mapping:

| `size` | Tailwind class |
|---|---|
| `sm` | `max-w-sm` |
| `md` | `max-w-md` |
| `lg` | `max-w-lg` |
| `xl` | `max-w-xl` |
| `2xl` | `max-w-2xl` |

### Slots

- **default** — body content. Forms (with their own submit/cancel buttons) live here.

### Emits

- **`close`** — fired when the user closes the dialog (ESC, backdrop click, or close X). Parent owns the open state via `v-if`.

### Built-in behaviour

- `<teleport to="body">` — always. Avoids parent stacking-context issues and keeps the dialog at the top of the DOM where focus management expects it.
- `role="dialog"` + `aria-modal="true"` + `aria-labelledby="<auto-generated id>"` on the panel.
- Backdrop: `bg-horizon-900/60` with `@click.self="$emit('close')"`.
- ESC key on the panel: `@keydown.esc="$emit('close')"`. The panel carries `tabindex="-1"` so it can receive keyboard events without being in the tab order.
- Close X button auto-injected in the header top-right (Tailwind: `text-horizon-300 hover:text-horizon-500 text-2xl leading-none`, `aria-label="Close dialog"`).
- Panel: `bg-white rounded-xl shadow-xl mx-4 max-h-[90vh] overflow-hidden flex flex-col`.
- Header: `px-6 py-4 border-b border-savannah-100 flex justify-between items-center` containing title + optional subtitle + close X.
- Body: `overflow-y-auto flex-1 px-6 py-4` so long forms scroll inside the panel rather than the page.

### Auto-generated id

Use Vue's `useId()` composable (project pins Vue 3.5+ in `package.json`) to give every instance a unique `aria-labelledby` target without prop drilling. The component is Options API to match the rest of the codebase, so import it as `import { useId } from 'vue'` inside `setup()` (or use `defineComponent`'s `setup()` block) and expose the id to the template.

## Architecture

### Component structure

```
DialogContainer.vue (single file, ~80 LOC)
├── <teleport to="body">
│   └── <div class="fixed inset-0 ... bg-horizon-900/60" @click.self="$emit('close')">
│       └── <div role="dialog" aria-modal="true" :aria-labelledby="titleId"
│                class="bg-white rounded-xl ... max-w-{size} max-h-[90vh] flex flex-col"
│                @keydown.esc="$emit('close')" tabindex="-1">
│           ├── <header class="px-6 py-4 border-b border-savannah-100 ...">
│           │   ├── <h2 :id="titleId" class="text-xl font-bold text-horizon-500">{{ title }}</h2>
│           │   ├── <p v-if="subtitle">{{ subtitle }}</p>
│           │   └── <button aria-label="Close dialog" @click="$emit('close')">×</button>
│           └── <div class="overflow-y-auto flex-1 px-6 py-4">
│               └── <slot />
```

### Refactor pattern (each modal)

Before:

```vue
<template>
  <div class="fixed inset-0 bg-horizon-900/60 ..." @click.self="$emit('close')">
    <div class="bg-white rounded-xl shadow-xl w-full max-w-lg p-6">
      <header class="mb-6">
        <h2 class="text-2xl font-black text-horizon-700">{{ pageTitle }}</h2>
        <p class="text-sm text-horizon-500 mt-1">{{ pageSubtitle }}</p>
      </header>
      <form @submit.prevent="handleSubmit" class="space-y-4">
        <!-- fields -->
      </form>
    </div>
  </div>
</template>
```

After:

```vue
<template>
  <DialogContainer :title="pageTitle" :subtitle="pageSubtitle" size="lg" @close="$emit('close')">
    <form @submit.prevent="handleSubmit" class="space-y-4">
      <!-- fields (unchanged) -->
    </form>
  </DialogContainer>
</template>
```

The form, validation logic, computed properties, and submission flow inside each modal are untouched.

## Migration order

Simplest first to surface API issues early. All 5 ship in one PR.

1. **`TfsaContributionModal.vue`** (Savings, `lg`) — currently no a11y, simplest body.
2. **`ZaInvestmentForm.vue`** (Investment, `lg`) — similar shape to TFSA.
3. **`ZaContributionModal.vue`** (Retirement, `lg`) — has a conditional subtitle line.
4. **`ZaRetirementFundForm.vue`** (Retirement, `xl` — taller form, internal scroll exercised).
5. **`ZaProtectionPolicyModal.vue`** (Protection, `2xl`) — already a11y-correct; refactor last because it tests that DialogContainer fully replaces the existing gold standard with no regression.

## Verification

For each of the 5 modals, in Playwright as `za-protection-test@example.com` on localhost:

1. Navigate to the parent page (`/za/savings`, `/za/investments`, `/za/retirement`, `/za/protection`).
2. Trigger the modal via its existing button.
3. Verify: panel rendered with correct title + subtitle, focus inside panel, body scroll locked behind backdrop.
4. Fill required fields; click submit; verify success state in the parent page.
5. Re-open the modal; press ESC; verify it closes without saving.
6. Re-open the modal; click the backdrop; verify it closes without saving.
7. Re-open the modal; click the close X; verify it closes without saving.

ZaProtectionPolicyModal additionally: verify `aria-labelledby` still points to the dynamic title (Add vs Edit), since it's the only modal using a dynamic title.

If any modal regresses, revert that modal's commit and continue. PR doesn't ship until all 5 are green.

## Risks & mitigations

| Risk | Mitigation |
|---|---|
| Slight backdrop visual change on ZaProtectionPolicyModal (`bg-horizon-500/40` → `bg-horizon-900/60`, slightly darker). | Approved upfront in design discussion. Document in PR description. |
| Form submission flow breaks if event listeners on the dialog interfere with form submit. | Each modal Playwright-tested for submit. DialogContainer doesn't intercept clicks/keys inside the body. |
| Focus trap not implemented (Vue/Tailwind don't provide one out of the box). | Out of scope — none of the 5 currently have a focus trap, so no regression. Add later as a separate enhancement if needed. |
| `<teleport to="body">` may surprise tests that assert DOM structure relative to the parent component. | None of the 5 have such tests. New per-modal Playwright tests query by accessible role, not DOM structure. |
| Dev DB lacks ZA jurisdiction for some test users. | `za-protection-test@example.com` already has full ZA jurisdiction; covers all 5 modules. |

## Out of scope (deferred follow-ups)

- Focus trap implementation.
- Initial-focus management (auto-focus first input).
- Animation (fade-in/scale-in on open).
- UK modal adoption.
- `Shared/ConfirmModal.vue` consolidation (different shape, stays separate).
- `Tabs.vue` extraction (deferred until 3rd tabs-bar example exists).
