## Summary

Phase 1 adds Progressive Web App (PWA) support, mobile-optimised API endpoints, responsive chat, and offline detection. This builds on Phase 0 (analytics + auth token abstraction).

**Key additions:**
- PWA manifest + service worker with Workbox caching strategies
- Mobile dashboard API (`/api/v1/mobile/dashboard`) — aggregated 6-module summary with net worth
- Module summary API (`/api/v1/mobile/modules/{module}`) — individual module analysis
- Daily insights API (`/api/v1/mobile/insights/daily`) — contextual Fyn tips
- ETag conditional caching middleware (304 Not Modified)
- Mobile-specific rate limiters (dashboard: 30/min, AI chat: 20/min, device registration: 5/min)
- Responsive full-screen chat overlay on mobile (<768px)
- QuickReplyChips for contextual chat suggestions
- 8 PWA icons (72px–512px)

## Pre-Deploy: Dependencies

## Rollback

If issues arise:
1. Re-upload the previous `public/build/` directory (removes PWA service worker)
2. Restore previous versions of `Kernel.php`, `RouteServiceProvider.php`, `api_v1.php`
3. The new controller/service/middleware files are inert without routes, but can be removed
4. Clear caches: `php artisan cache:clear && php artisan config:clear && php artisan route:clear && php artisan optimize`
