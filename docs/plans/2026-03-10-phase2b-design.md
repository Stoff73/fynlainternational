# Phase 2b Design — Capacitor iOS App

**Date:** 10 March 2026
**Status:** Approved
**Branch:** `feature/mobile-app-phase0`
**Prerequisite:** Phase 2a complete (all backend infrastructure deployed)

---

## Goal

Build a complete iOS app using Capacitor 6.x wrapping the existing Vue.js frontend, with native biometric auth, push notifications, voice input, haptic feedback, and a mobile-optimised UI with 5-tab navigation.

## Architecture

Hybrid approach: Capacitor shell provides native capabilities (biometrics, push, haptics, voice, keychain), while the UI is Vue.js components rendered in a WKWebView. Most UI development is browser-testable via `./dev.sh`; native features tested via `npx cap run ios`. All API calls go to the existing Laravel backend at `https://fynla.org`.

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Platform | iOS only | Ship faster, add Android later |
| TestFlight strategy | Full build then submit | Complete app on first device |
| Biometrics | Face ID + Touch ID | Plugin handles both transparently |
| Voice input | Included | Full Fyn chat experience from day one |
| Haptics | Included | Native feel from day one |
| Learn Hub | In-app with Fyn-led experience | Info popups, icons, helper text, external links |
| Module summaries | All 7, simplified | Hero metric + "View full detail on web" |
| Push registration | Contextual | Higher opt-in, Apple-recommended |
| Mortgage alerts | Fixed rate expiry at 90/60/30 days | Uses existing mortgage data |
| App icons source | `public/images/logos/favicon.png` | Single source of truth |

## Section 2: Mobile Auth Flow

**First login (no stored credentials):**
1. App opens -> no token in `@capacitor/preferences` -> show login screen
2. Email + password -> `POST /api/v1/auth/login`
3. Verification code -> `POST /api/v1/auth/verify-code` -> 30-day Sanctum token
4. Token stored via `tokenStorage.js` (already abstracts `@capacitor/preferences` on native)
5. Prompt: "Enable Face ID / Touch ID for faster login?" -> if yes, store biometric flag
6. Register device: `POST /api/v1/mobile/devices` (existing Phase 2a endpoint)
7. Navigate to dashboard

**Subsequent launches:**
1. Token found -> biometric prompt (Face ID or Touch ID)
2. Success -> retrieve token -> `GET /api/v1/auth/user` to validate
3. Valid -> dashboard. 401 -> clear token, show login.

**Token refresh:**
1. Each launch, check token age
2. If >25 days -> `POST /api/v1/auth/refresh-token` (existing) -> new token, old revoked
3. If >30 days (expired) -> full re-auth

**Biometric implementation:**
```javascript
import { NativeBiometric } from '@capgo/capacitor-native-biometric';

const { isAvailable } = await NativeBiometric.isAvailable();

// Store after first login
await NativeBiometric.setCredentials({
  username: email, password: token, server: 'fynla.org',
});

// Retrieve on subsequent launch
const credentials = await NativeBiometric.getCredentials({ server: 'fynla.org' });
```

**App lifecycle:**
```javascript
import { App } from '@capacitor/app';
App.addListener('appStateChange', ({ isActive }) => {
  if (!isActive) store.dispatch('aiChat/abortStreaming');
  else {
    store.dispatch('auth/validateToken');
    store.dispatch('mobileDashboard/refreshDashboard');
  }
});
```

**New components:**
- `MobileLoginScreen.vue` — email/password form, keyboard-aware
- `BiometricPrompt.vue` — Face ID/Touch ID enable prompt + re-auth
- `VerificationCodeScreen.vue` — 6-digit code input, auto-advance

## Section 9: Push Notifications Frontend

**Contextual registration** — no upfront prompt.

**Trigger points:**

| Moment | Prompt text |
|--------|------------|
| First goal created | "Want Fyn to notify you when you hit milestones?" |
| First policy with upcoming renewal | "Want reminders when your policies are due for renewal?" |
| First mortgage with expiring fixed rate | "Want alerts before your fixed rate expires?" |
| Notification settings opened | "Enable notifications to use these preferences" |

**Flow:**
1. Soft prompt (in-app bottom sheet) explaining value
2. "Enable" -> iOS system permission dialog
3. Granted -> register token with `POST /api/v1/mobile/devices`
4. Denied -> respect, show toggle in settings to retry (opens iOS Settings)

**Foreground notifications:** In-app toast banner (slides down, auto-dismiss 4 seconds).

**Notification tap deep links:**

| Type | Opens |
|------|-------|
| Policy renewal | `/m/more/summary/protection` |
| Goal milestone | `/m/goals/:id` |
| Contribution reminder | `/m/goals` |
| Mortgage rate alert | `/m/more/summary/savings` |
| Security alert | web via SFSafariVC |
| Daily insight | `/m/home` |
| Payment/subscription | web via SFSafariVC |

**New Vuex store — `mobileNotifications.js`:**
- State: `permissionStatus`, `unreadCount`, `inAppNotification`
- Actions: `requestPermission`, `registerToken`, `showInAppNotification`, `clearUnread`
- Persisted: `permissionStatus` only

**Components:**
- `PushPermissionPrompt.vue` — `resources/js/mobile/PushPermissionPrompt.vue`
- `InAppNotificationToast.vue` — `resources/js/mobile/InAppNotificationToast.vue`
- `mobileNotifications.js` — `resources/js/store/modules/mobileNotifications.js`

## Section 10: Mortgage Rate Alerts (New)

**Backend additions to Phase 2a infrastructure:**

- **Migration:** Add `mortgage_rate_alerts` boolean column to `notification_preferences` (default `true`)
- **Scheduled command:** `SendMortgageRateAlerts.php` — daily at 09:30, queries mortgages with `fixed_rate_end_date` at 90/60/30 days out
- **Notification class:** `MortgageRateAlertNotification.php`
- **Notification text:** "Your fixed rate on {property address} expires in {days} days. Now might be a good time to review your options."
- **Schedule:** `$schedule->command('notifications:mortgage-rate-alerts')->dailyAt('09:30');`
- **Update:** `NotificationPreferenceController` show response + `UpdateNotificationPreferencesRequest` to include `mortgage_rate_alerts`

## Section 11: Build Pipeline & iOS Configuration

**Build script — `deploy/mobile/build-ios.sh`:**
```bash
#!/bin/bash
set -e
echo "Building Fynla iOS app..."
export VITE_BASE_PATH=/
export VITE_API_BASE_URL=https://fynla.org
export VITE_PLATFORM=ios
npm run build
npx cap sync ios
echo "Done. Open ios/App/App.xcworkspace in Xcode."
```

**Xcode configuration:**
- Bundle ID: `org.fynla.app`
- Deployment target: iOS 16.0
- Signing: automatic with Apple Developer account
- Capabilities: Push Notifications, Associated Domains (`applinks:fynla.org`), Keychain Sharing

**Info.plist permissions:**

| Key | String | For |
|-----|--------|-----|
| `NSMicrophoneUsageDescription` | "Fynla uses your microphone for voice input when chatting with Fyn" | Voice |
| `NSFaceIDUsageDescription` | "Fynla uses Face ID for quick, secure login" | Biometrics |

**App icons:** Generated from `public/images/logos/favicon.png`. Sizes: 20pt, 29pt, 40pt, 60pt, 76pt, 83.5pt, 1024pt (@2x and @3x).

**Splash screen:** `bg-eggshell-500`, centred Fynla logo, raspberry-500 spinner.

**Development workflow:**
1. `./dev.sh` for browser-based UI work at `localhost:5173/m/home`
2. `npx cap run ios` for native features on simulator/device
3. `deploy/mobile/build-ios.sh` -> Xcode archive -> TestFlight
