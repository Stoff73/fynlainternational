## 1. ProtectionWorkflowTest — 4 failures fixed

Root cause was **NOT** Rule 13 / banned `adequacy_score` (CSJTODO description was misleading). Three actual issues:

- **Readiness gate added.** `ProtectionAgent::analyze()` now invokes `ProtectionDataReadinessService::assess()` and returns `success: true, message: 'Readiness check incomplete', data: { can_proceed: false, ... }` when blocking checks fail. The blocking checks are `date_of_birth`, `income` (User-level: `annual_employment_income` etc.), and `marital_status`. Test fixtures only set DOB; income lived on `ProtectionProfile.annual_income` (not the User model), so the gate refused to analyze.
- **`adequacy_score` shape changed from scalar to array.** Now returns `['overall_score' => int, 'rating' => string, 'color' => string, 'insights' => [...], 'life_insurance_score', 'critical_illness_score', 'income_protection_score']`. Tests asserted `> 0` against the array.
- **TaxConfigurationSeeder not auto-seeded for `tests/Integration/`.** The `tests/Pest.php` `beforeEach` factory hook covers `Feature` and `Unit/Services`, but `TaxConfigService::loadActiveConfig()` still threw inside the analyze flow — easier to add explicit `$this->seed(TaxConfigurationSeeder::class)` in this file's `beforeEach`, matching the Feature-test pattern.
- **Soft-delete assertion.** `CriticalIllnessPolicy` uses `SoftDeletes`; `assertDatabaseMissing` is wrong → switched to `assertSoftDeleted`.

Fixes in `tests/Integration/ProtectionWorkflowTest.php`:
1. Added `beforeEach` seeding `TaxConfigurationSeeder`.
2. Added `annual_employment_income` to User factory creates on tests 1, 4, 5.
3. Updated test 1 + test 4 to assert the array shape (`overall_score`, `rating`).
4. Updated test 3 to assert the new readiness response (`success: true, can_proceed: false`).
5. Updated test 5 to compare `data.gaps.gaps_by_category` instead of relying on score deltas.
6. Replaced `assertDatabaseMissing` with `assertSoftDeleted`.
