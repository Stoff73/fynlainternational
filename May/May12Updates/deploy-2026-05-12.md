## Rollback plan

If prod breaks:

1. Roll back the deploy by uploading the previous `public/build/` artefact + restoring PHP files from the previous deploy backup
2. The migration is **one-way** (no `down()`) — but the new rows are still valid for both class names while the `class_alias` is in place, so a code rollback alone fixes auth without needing a DB rollback
3. The 370 rows will now read `Fynla\Core\Models\User`; if rolled-back code lacks the class, Sanctum will throw `ClassNotFoundException`. Add a temp `class_alias('Fynla\\Core\\Models\\User', 'App\\Models\\User')` in the old `app/Models/User.php` (reverse direction) if needed during recovery
